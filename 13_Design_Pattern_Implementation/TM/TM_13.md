## Apa Itu Event Delegation?

**Event Delegation** adalah sebuah pola desain (_design pattern_) dalam JavaScript di mana kita **mendaftarkan satu event listener pada elemen induk (_parent_)**, alih-alih mendaftarkan banyak event listener pada masing-masing elemen anak (_children_).

Pola ini memanfaatkan mekanisme alami browser yang disebut **event bubbling** — di mana sebuah event yang terjadi pada elemen anak akan merambat (_bubble up_) ke atas menuju elemen-elemen induknya hingga mencapai `document`.

```
document
    └── <ul id="menu">           ← 1 listener di sini
            ├── <li>Home</li>    ← tidak perlu listener
            ├── <li>About</li>   ← tidak perlu listener
            └── <li>Contact</li> ← tidak perlu listener
```

Dengan satu listener di `<ul>`, kita bisa menangkap klik pada **semua** `<li>` di dalamnya — bahkan `<li>` yang belum ada saat halaman dimuat sekalipun.

## Prasyarat: Memahami Event Bubbling

Sebelum memahami event delegation, kita harus mengerti terlebih dahulu bagaimana event mengalir di DOM.

### Tiga Fase Perjalanan Event

Ketika pengguna mengklik sebuah elemen, event tidak langsung "mati" di situ. Event tersebut menjalani tiga fase:

```
                    ┌─────────────────────────────┐
  FASE 1            │          document            │ ──┐
  (Capture)         ├─────────────────────────────┤   │ Event
                    │           <html>             │   │ turun
                    ├─────────────────────────────┤   ↓ ke target
                    │           <body>             │
                    ├─────────────────────────────┤
  FASE 2            │            <ul>              │ ← TARGET ANCESTOR
  (Target)          ├─────────────────────────────┤
                    │     <li> ← USER CLICKS HERE  │ ← TARGET ELEMENT
                    └─────────────────────────────┘
                    ┌─────────────────────────────┐
  FASE 3            │            <ul>              │   ↑ Event
  (Bubble)          ├─────────────────────────────┤   │ naik
                    │           <body>             │   │ ke atas
                    ├─────────────────────────────┤   │
                    │           <html>             │ ──┘
                    ├─────────────────────────────┤
                    │          document            │
                    └─────────────────────────────┘
```

| Fase | Nama        | Arah         | Keterangan                               |
| ---- | ----------- | ------------ | ---------------------------------------- |
| 1    | **Capture** | Atas → Bawah | Event merambat dari `document` ke target |
| 2    | **Target**  | —            | Event tiba di elemen yang diklik         |
| 3    | **Bubble**  | Bawah → Atas | Event merambat kembali ke `document`     |

> Event Delegation mengeksploitasi **Fase 3 (Bubble)**. Karena event naik ke atas, elemen induk pun dapat "mendengar" event yang terjadi di anaknya.

### Membuktikan Bubbling

```javascript
document.querySelector("ul").addEventListener("click", function (event) {
  // Ini akan terpanggil meskipun yang diklik adalah <li> di dalamnya!
  console.log("ul mendengar klik dari:", event.target.tagName);
});
```

## Masalah yang Dipecahkan

Bayangkan sebuah aplikasi todo list. Setiap item memiliki tombol hapus.

### Pendekatan Naif (Tanpa Delegation)

```javascript
// Anggap ada 1000 item todo
const deleteButtons = document.querySelectorAll(".delete-btn");

deleteButtons.forEach((button) => {
  button.addEventListener("click", function () {
    this.parentElement.remove();
  });
});
```

**Masalah pendekatan ini:**

1. **Memori membengkak** — 1000 item = 1000 event listener tersimpan di memori.
2. **Item baru tidak ter-cover** — Jika pengguna menambah item baru via JavaScript, tombol hapusnya tidak akan berfungsi karena `querySelectorAll` hanya menangkap elemen yang _sudah ada_ saat kode dijalankan.
3. **Kode repetitif** — Kita harus me-loop dan mendaftarkan listener satu per satu.

```javascript
// Ini TIDAK AKAN BEKERJA untuk item yang ditambahkan dinamis:
const newTodo = document.createElement("li");
newTodo.innerHTML =
  '<span>Item baru</span><button class="delete-btn">Hapus</button>';
document.querySelector("#todo-list").appendChild(newTodo);
// Tombol "Hapus" di item baru ini tidak memiliki listener!
```

## Anatomi Event Delegation

```javascript
//         1. Elemen induk (parent)
//         ↓
parentElement.addEventListener("click", function (event) {
  //                                              ↑
  //                              2. Event object yang membawa info
  //                                 tentang SIAPA yang diklik

  //  3. Cek apakah elemen yang diklik adalah yang kita inginkan
  if (event.target.matches(".delete-btn")) {
    //         ↑
    //    event.target = elemen SESUNGGUHNYA yang diklik pengguna

    // 4. Lakukan aksi
    event.target.closest("li").remove();
  }
});
```

### Tiga Komponen Kunci

| Komponen              | Peran                                                  |
| --------------------- | ------------------------------------------------------ |
| `parentElement`       | Elemen "penjaga gerbang" yang menampung semua listener |
| `event.target`        | Elemen spesifik yang benar-benar diklik pengguna       |
| `event.currentTarget` | Elemen di mana listener didaftarkan (si parent)        |

## Implementasi: Dari Naive ke Delegation

### Studi Kasus: Todo List Dinamis

#### Versi Naif

```html
<ul id="todo-list">
  <li>Beli sayuran <button class="delete-btn">✕</button></li>
  <li>Cuci motor <button class="delete-btn">✕</button></li>
</ul>
<input id="new-todo" placeholder="Tambah todo..." />
<button id="add-btn">Tambah</button>
```

```javascript
// PENDEKATAN NAIF
function attachDeleteListeners() {
  // Harus dipanggil ulang setiap kali item baru ditambahkan!
  document.querySelectorAll(".delete-btn").forEach((btn) => {
    btn.addEventListener("click", function () {
      this.parentElement.remove();
    });
  });
}

document.getElementById("add-btn").addEventListener("click", function () {
  const text = document.getElementById("new-todo").value;
  const li = document.createElement("li");
  li.innerHTML = `${text} <button class="delete-btn">✕</button>`;
  document.getElementById("todo-list").appendChild(li);

  // Harus memanggil fungsi ini lagi — rawan lupa dan bug!
  attachDeleteListeners();
});

attachDeleteListeners();
```

#### Versi dengan Event Delegation

```javascript
// PENDEKATAN EVENT DELEGATION
const todoList = document.getElementById("todo-list");

// Cukup satu listener, sekali saja, selamanya.
todoList.addEventListener("click", function (event) {
  if (event.target.matches(".delete-btn")) {
    event.target.closest("li").remove();
  }
});

document.getElementById("add-btn").addEventListener("click", function () {
  const text = document.getElementById("new-todo").value;
  const li = document.createElement("li");
  li.innerHTML = `${text} <button class="delete-btn">✕</button>`;
  todoList.appendChild(li);
  // Item baru langsung bekerja! Tidak perlu setup listener tambahan.
});
```

Jauh lebih bersih, efisien, dan _future-proof_.

## Menggunakan event.target dengan Bijak

### `event.target` vs `event.currentTarget`

Ini adalah sumber kebingungan terbesar bagi pemula. Mari kita bedah:

```html
<ul id="menu">
  <!-- currentTarget (tempat listener didaftarkan) -->
  <li class="menu-item">
    <span class="icon">🏠</span>
    <span class="label">Home</span>
    <!-- target (yang mungkin diklik) -->
  </li>
</ul>
```

```javascript
document.getElementById("menu").addEventListener("click", function (event) {
  console.log("target:", event.target); // Bisa <span>, bisa <li>
  console.log("currentTarget:", event.currentTarget); // Selalu <ul>
});
```

**Masalah:** Jika pengguna mengklik teks "Home" (elemen `<span>`), maka `event.target` adalah `<span>`, bukan `<li>`. Padahal kita ingin mendeteksi klik pada `<li>`.

### Solusi: `closest()`

Metode `closest()` mencari elemen terdekat ke atas (ancestor) yang cocok dengan selector. Ini adalah pasangan sempurna untuk event delegation.

```javascript
document.getElementById("menu").addEventListener("click", function (event) {
  // Cari <li> terdekat dari elemen yang diklik
  const menuItem = event.target.closest(".menu-item");

  // Jika klik terjadi di luar .menu-item, closest() mengembalikan null
  if (!menuItem) return;

  // Sekarang kita aman bekerja dengan menuItem
  console.log("Item diklik:", menuItem.textContent);
});
```

### Perbandingan `matches()` vs `closest()`

| Metode                         | Deskripsi                         | Kapan Digunakan                                                                |
| ------------------------------ | --------------------------------- | ------------------------------------------------------------------------------ |
| `event.target.matches('.btn')` | Cek apakah _target_ persis cocok  | Struktur HTML sederhana, tidak ada elemen anak di dalam target                 |
| `event.target.closest('.btn')` | Cari ancestor terdekat yang cocok | Struktur HTML kompleks, target bisa berupa elemen anak dari yang kita inginkan |

```javascript
// Aman untuk struktur apapun:
parentEl.addEventListener("click", function (event) {
  const btn = event.target.closest(".action-btn");
  if (btn && this.contains(btn)) {
    // this.contains(btn) memastikan btn masih di dalam parentEl
    // (penting jika parentEl bukan document)
    handleAction(btn);
  }
});
```

## Pola Lanjutan

### Pola 1: Multi-Action Delegation

Gunakan satu listener untuk menangani beberapa jenis aksi berbeda dengan atribut `data-*`.

```html
<ul id="user-list">
  <li data-id="1">
    Budi
    <button data-action="edit" data-id="1">Edit</button>
    <button data-action="delete" data-id="1">Hapus</button>
    <button data-action="promote" data-id="1">Promosi</button>
  </li>
</ul>
```

```javascript
const actions = {
  edit(id) {
    console.log(`Mengedit user ${id}`);
  },
  delete(id) {
    console.log(`Menghapus user ${id}`);
  },
  promote(id) {
    console.log(`Mempromosikan user ${id}`);
  },
};

document
  .getElementById("user-list")
  .addEventListener("click", function (event) {
    const btn = event.target.closest("[data-action]");
    if (!btn) return;

    const { action, id } = btn.dataset;

    if (actions[action]) {
      actions[action](id);
    }
  });
```

Pola ini sangat _scalable_ — menambah aksi baru cukup dengan menambah metode ke objek `actions` dan atribut `data-action` di HTML.

### Pola 2: Delegation Factory (Reusable)

Buat fungsi utilitas yang bisa dipakai di mana saja:

```javascript
/**
 * Mendaftarkan event delegation pada sebuah elemen.
 * @param {Element} parent - Elemen induk tempat listener didaftarkan
 * @param {string} eventType - Tipe event ('click', 'input', dll)
 * @param {string} selector - CSS selector untuk elemen target
 * @param {Function} handler - Fungsi yang dipanggil saat event terjadi
 */
function delegate(parent, eventType, selector, handler) {
  parent.addEventListener(eventType, function (event) {
    const target = event.target.closest(selector);
    if (target && parent.contains(target)) {
      handler.call(target, event, target);
    }
  });
}

// Penggunaan:
delegate(
  document.getElementById("todo-list"),
  "click",
  ".delete-btn",
  function (event) {
    this.closest("li").remove(); // `this` merujuk ke .delete-btn
  },
);

delegate(
  document.getElementById("form-container"),
  "change",
  'input[type="checkbox"]',
  function (event, checkbox) {
    console.log("Checkbox berubah:", checkbox.name, checkbox.checked);
  },
);
```

### Pola 3: Delegation pada Document Level

Untuk aplikasi berskala besar (seperti SPA), kadang delegation langsung pada `document` lebih praktis:

```javascript
// Satu listener untuk seluruh halaman
document.addEventListener("click", function (event) {
  // Modal trigger
  const modalTrigger = event.target.closest("[data-modal]");
  if (modalTrigger) {
    openModal(modalTrigger.dataset.modal);
    return;
  }

  // Dropdown toggle
  const dropdownToggle = event.target.closest("[data-dropdown]");
  if (dropdownToggle) {
    toggleDropdown(dropdownToggle.dataset.dropdown);
    return;
  }

  // Tutup dropdown jika klik di luar
  if (!event.target.closest(".dropdown")) {
    closeAllDropdowns();
  }
});
```

> Hati-hati mendaftarkan terlalu banyak logika di `document`. Pertimbangkan untuk memecahnya ke beberapa parent yang lebih spesifik demi keterbacaan kode.

### Pola 4: Delegation dengan `stopPropagation` yang Aman

Terkadang ada elemen anak yang _tidak boleh_ mentrigger handler dari delegation. Gunakan pengecekan eksplisit:

```javascript
parentEl.addEventListener("click", function (event) {
  // Abaikan klik pada elemen .no-delegate dan turunannya
  if (event.target.closest(".no-delegate")) return;

  const item = event.target.closest(".item");
  if (item) {
    handleItemClick(item);
  }
});
```

Lebih aman daripada `stopPropagation()` karena tidak memutus chain event untuk listener lain.

## Kapan Menggunakan & Kapan Tidak

### Gunakan Event Delegation Ketika...

- **Daftar atau tabel dengan banyak item** — todo list, tabel data, galeri gambar
- **Konten yang dirender secara dinamis** — hasil fetch API, item yang ditambah/dihapus oleh pengguna
- **Komponen berulang** — kartu produk, notifikasi, item menu
- **Performa kritikal** — aplikasi dengan ratusan atau ribuan elemen interaktif

### Hindari Event Delegation Ketika...

- **Elemen tidak memiliki parent yang stabil** — jika parent juga diganti-ganti secara dinamis
- **Event tidak mendukung bubbling** — beberapa event seperti `focus`, `blur`, dan `mouseenter` tidak bubble secara default (gunakan `focusin`/`focusout` sebagai penggantinya)
- **Satu elemen tunggal yang statis** — `document.getElementById('submit-btn')` sudah cukup; menggunakan delegation justru _overengineering_
- **Membutuhkan `event.stopPropagation()`** secara agresif — ini akan merusak chain delegation

### Event yang Tidak Mendukung Bubbling

```javascript
// Ini TIDAK AKAN BEKERJA — blur tidak bubble
parentEl.addEventListener("blur", function (event) {
  console.log("input kehilangan fokus"); // Tidak terpanggil!
});

// Gunakan focusout sebagai pengganti — focusout bubble
parentEl.addEventListener("focusout", function (event) {
  const input = event.target.closest("input");
  if (input) {
    validateInput(input);
  }
});

// Daftar event yang TIDAK bubble:
// blur, focus, mouseenter, mouseleave, load, unload, scroll (pada elemen)
// Gantinya: focusout, focusin, mouseover, mouseout
```

## Gotchas & Jebakan Umum

### Jebakan 1: Lupa Memfilter Target

```javascript
// BUG: Akan terpanggil untuk SEMUA klik di dalam ul,
//         bahkan klik pada area kosong
ul.addEventListener("click", function (event) {
  handleItemClick(event.target); // event.target mungkin <ul> itu sendiri!
});

// BENAR: Selalu filter target
ul.addEventListener("click", function (event) {
  const item = event.target.closest("li");
  if (!item) return; // Keluar jika bukan li
  handleItemClick(item);
});
```

### Jebakan 2: Delegation di Luar Konteks Parent

```javascript
const container = document.getElementById("container");

container.addEventListener("click", function (event) {
  const btn = event.target.closest(".btn");

  // BAHAYA: closest() bisa "melarikan diri" ke luar container!
  // Jika .btn ada di luar #container, ia tetap akan tertangkap!
  if (btn) {
    handleClick(btn); // Mungkin menangani elemen di luar container
  }

  // AMAN: Verifikasi btn masih di dalam container
  if (btn && container.contains(btn)) {
    handleClick(btn);
  }
});
```

### Jebakan 3: Mengabaikan `this` dalam Arrow Function

```javascript
// Arrow function: `this` tidak merujuk ke currentTarget
parentEl.addEventListener("click", (event) => {
  console.log(this); // undefined (atau window di non-strict mode)
});

// Regular function: `this` merujuk ke currentTarget (parentEl)
parentEl.addEventListener("click", function (event) {
  console.log(this); // parentEl
  console.log(this === event.currentTarget); // true
});
```

### Jebakan 4: Memory Leak pada Delegation yang Tidak Dilepas

```javascript
// Jika sebuah komponen dihancurkan, pastikan listener ikut dilepas
class TodoComponent {
  constructor(container) {
    this.container = container;
    this.handleClick = this.handleClick.bind(this); // Simpan referensi
    this.container.addEventListener("click", this.handleClick);
  }

  handleClick(event) {
    const deleteBtn = event.target.closest(".delete-btn");
    if (deleteBtn) {
      deleteBtn.closest("li").remove();
    }
  }

  destroy() {
    // Lepas listener saat komponen dihancurkan
    this.container.removeEventListener("click", this.handleClick);
  }
}
```

## Perbandingan Performa

### Skenario: 10.000 Item List

| Aspek                 | Tanpa Delegation              | Dengan Delegation        |
| --------------------- | ----------------------------- | ------------------------ |
| Jumlah Event Listener | 10.000                        | 1                        |
| Penggunaan Memori     | Tinggi (~O(n))                | Konstan (~O(1))          |
| Waktu Setup           | Lambat (loop 10k)             | Instan                   |
| Item Dinamis Baru     | Perlu setup ulang             | Langsung bekerja         |
| Pembersihan (Cleanup) | Kompleks (lepas 10k listener) | Mudah (lepas 1 listener) |

### Simulasi Penggunaan Memori

```javascript
// Tanpa Delegation
// Memori: ~10.000 closure objects, masing-masing menyimpan referensi ke elemen
document.querySelectorAll(".item").forEach((item) => {
  item.addEventListener("click", function () {
    /* closure */
  });
  // ↑ Setiap listener adalah objek tersendiri di heap
});

// Dengan Delegation
// Memori: 1 closure object saja
document.getElementById("list").addEventListener("click", function (event) {
  /* 1 closure tunggal untuk semua item */
});
```

> **Catatan:** Untuk jumlah elemen kecil (< 100), perbedaan performa tidak signifikan secara praktis. Pilih delegation bukan hanya karena performa, tapi juga karena **kemudahan menangani elemen dinamis**.

## Rangkuman

Event delegation adalah salah satu pola paling elegan dalam JavaScript karena ia memanfaatkan perilaku alami browser (event bubbling) untuk menyelesaikan tiga masalah sekaligus:

```
┌─────────────────────────────────────────────────────────────────┐
│                    EVENT DELEGATION                              │
│                                                                  │
│  PRINSIP: Satu listener di parent → menangkap event dari semua  │
│           children, termasuk yang baru ditambahkan dinamis       │
│                                                                  │
│  MEMANFAATKAN: Event Bubbling                                    │
│  (Event merambat dari child ke parent secara alami)              │
│                                                                  │
│  ALAT UTAMA:                                                     │
│  ┌──────────────────────┬───────────────────────────────────┐   │
│  │ event.target         │ Elemen yang diklik pengguna        │   │
│  │ event.currentTarget  │ Elemen tempat listener didaftarkan │   │
│  │ .closest(selector)   │ Cari ancestor yang cocok           │   │
│  │ .matches(selector)   │ Cek kecocokan tepat                │   │
│  │ .contains(element)   │ Validasi masih dalam parent        │   │
│  └──────────────────────┴───────────────────────────────────┘   │
│                                                                  │
│  HASIL:                                                          │
│  Memori lebih efisien                                            │
│  Bekerja untuk elemen dinamis                                    │
│  Kode lebih bersih dan maintainable                              │
└─────────────────────────────────────────────────────────────────┘
```

### Resep Singkat Event Delegation

```javascript
// 1. Pilih parent yang tepat dan stabil
const parent = document.getElementById("container");

// 2. Daftarkan listener di parent
parent.addEventListener("click", function (event) {
  // 3. Temukan elemen target yang relevan
  const target = event.target.closest(".item-btn");

  // 4. Validasi target ada dan masih di dalam parent
  if (!target || !parent.contains(target)) return;

  // 5. Ambil data dan eksekusi aksi
  const { id, action } = target.dataset;
  performAction(action, id);
});
```
